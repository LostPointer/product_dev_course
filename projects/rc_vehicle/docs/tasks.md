# Задачи на интеграцию датчиков

## TASK-RC-001 — Интеграция LSM6DS3 (6‑DOF IMU)
**Приоритет:** HIGH
**Статус:** [ ] В пути (ожидается 12-15 апреля)

**Цель:** использовать LSM6DS3 как основной IMU (акселерометр + гироскоп) для измерения ускорений и угловых скоростей.

**Описание:**
LSM6DS3 — это 6‑DOF IMU (акселерометр + гироскоп) от STMicroelectronics. Обеспечивает высокую частоту обновления и низкий шум.

**Характеристики:**
- Акселерометр: ±2/±4/±8/±16g, низкий шум
- Гироскоп: ±250/±500/±1000/±2000 dps
- Частота обновления: до 1.6 кГц (акселерометр), до 1 кГц (гироскоп)
- Встроенный температурный датчик
- Интерфейс: I2C или SPI (3.3V логика)
- Напряжение: 1.71–3.6V (требуется 3.3V питание)

**Интеграция позволит:**
- Измерять ускорения и угловые скорости для телеметрии и систем управления
- Использовать как замену MPU‑6500
- (Опционально) Добавить отдельный магнитометр (MMC5983MA) для компенсации yaw drift

**Требуемые работы:**

1. **Аппаратная часть:**
   - [ ] Подключить LSM6DS3 к ESP32‑S3 по I2C или SPI (предпочтительно SPI для высокой частоты)
   - [ ] Разместить датчик как можно дальше от силовых проводов/мотора (минимизировать вибрации и помехи)
   - [ ] Демонтировать MPU‑6500 (если LSM6DS3 используется как замена)
   - [ ] Добавить в BOM (см. `docs/bom_ru.md`)

2. **Драйвер датчика:**
   - [ ] Создать класс `Lsm6ds3Driver` для ESP32‑S3
   - [ ] Реализовать инициализацию (акселерометр + гироскоп)
   - [ ] Чтение сырых данных (accel_x/y/z, gyro_x/y/z)
   - [ ] Добавить калибровку смещения (bias для акселерометра/гироскопа)
   - [ ] Реализовать проверку связи (WHO_AM_I — уточнить в даташите)

3. **Сенсорная фузия:**
   - [ ] Использовать `MadgwickFilter` для оценки ориентации (6-DOF AHRS)
   - [ ] (Опционально) Добавить поддержку магнитометра если будет установлен MMC5983MA

4. **Калибровка:**
   - [ ] Калибровка акселерометра: статическое положение (6 граней)
   - [ ] Калибровка гироскопа: статическое положение (сбор данных в покое)
   - [ ] Сохранение калибровочных коэффициентов в NVS
   - [ ] WS-команда `calibrate_imu` с обратной связью в UI

5. **Телеметрия:**
   - [ ] Добавить в `TelemetryLogFrame`: `accel_x/y/z`, `gyro_x/y/z`
   - [ ] Отправка через WebSocket в реальном времени

6. **Тестирование:**
   - [ ] Unit-тесты: драйвер LSM6DS3, калибровка
   - [ ] Интеграционный тест: сравнение с MPU‑6500 (если оба доступны)
   - [ ] Полевой тест: стабильность данных при движении

**Зависимости:**
- Требуется поставка LSM6DS3
- Не блокирует базовую функциональность (можно работать с MPU‑6500)

**Файлы для модификации:**
- `projects/rc_vehicle/firmware/common/lsm6ds3_driver.hpp` (новый)
- `projects/rc_vehicle/firmware/common/lsm6ds3_driver.cpp` (новый)
- `projects/rc_vehicle/firmware/common/madgwick_filter.hpp`
- `projects/rc_vehicle/firmware/common/vehicle_ekf.hpp`
- `projects/rc_vehicle/firmware/common/telemetry_log.hpp`
- `projects/rc_vehicle/firmware/esp32_common/imu_handler.cpp`

**Оценка сложности:** 3–5 дней (включая отладку на реальном устройстве)

**Примечание:** LSM6DS3 — это 6-DOF IMU без магнитометра. Для компенсации дрейфа yaw потребуется отдельный магнитометр (MMC5983MA) или использование ICM‑20948 (9-DOF).

## TASK-RC-002 — Интеграция ICM‑20948 (9‑DOF IMU)
**Приоритет:** HIGH
**Статус:** [ ] В пути (ожидается 29 апреля - 2 мая)

**Цель:** использовать ICM‑20948 как основной 9‑DOF IMU (акселерометр + гироскоп + магнитометр) для полной ориентации и абсолютного heading.

**Описание:**
ICM‑20948 — это 9‑DOF IMU (акселерометр + гироскоп + магнитометр) в одном корпусе. Это упрощает монтаж (один датчик вместо двух) и обеспечивает лучшую синхронизацию данных.

**Характеристики:**
- Акселерометр: ±2/±4/±8/±16g, низкий шум
- Гироскоп: ±250/±500/±1000/±2000 dps, низкий дрейф
- Магнитометр: 3-осевой, ±4900 µT
- Частота обновления: до 1.1 кГц (акселерометр/гироскоп), 100 Гц (магнитометр)
- Интерфейс: I2C (адрес 0x68/0x69) или SPI (3.3V логика)
- Напряжение: 1.71–3.45V (требуется 3.3V питание)

**Интеграция позволит:**
- Компенсировать дрейф yaw из Madgwick/AHRS (сейчас yaw дрейфует без магнитометра)
- Получать абсолютный heading (курс относительно магнитного севера)
- Упростить монтаж (один датчик вместо LSM6DS3/MMC5983MA)
- Улучшить синхронизацию данных IMU (все сенсоры в одном чипе)

**Требуемые работы:**

1. **Аппаратная часть:**
   - [ ] Подключить ICM‑20948 к ESP32‑S3 по I2C или SPI (предпочтительно SPI для высокой частоты)
   - [ ] Разместить датчик как можно дальше от силовых проводов/мотора (минимизировать магнитные помехи)
   - [ ] Демонтировать MPU‑6500/LSM6DS3 (если ICM‑20948 используется как замена)
   - [ ] Добавить в BOM (см. `docs/bom_ru.md`)

2. **Драйвер датчика:**
   - [ ] Создать класс `Icm20948Driver` для ESP32‑S3
   - [ ] Реализовать инициализацию (акселерометр + гироскоп + магнитометр)
   - [ ] Чтение сырых данных (accel_x/y/z, gyro_x/y/z, mag_x/y/z)
   - [ ] Добавить калибровку смещения (hard-iron для магнитометра, bias для акселерометра/гироскопа)
   - [ ] Реализовать проверку связи (WHO_AM_I = 0xEA)

3. **Сенсорная фузия:**
   - [ ] Расширить `MadgwickFilter` для работы с магнитометром (9-DOF AHRS)
   - [ ] Альтернатива: встроенный DMP (Digital Motion Processor) в ICM‑20948 (если есть поддержка)
   - [ ] Добавить параметр `use_magnetometer` для переключения режимов 6-DOF / 9-DOF

4. **Калибровка:**
   - [ ] Процедура калибровки магнитометра: вращение машины на 360° в горизонтальной плоскости
   - [ ] Калибровка акселерометра: статическое положение (6 граней)
   - [ ] Сохранение калибровочных коэффициентов в NVS
   - [ ] WS-команда `calibrate_imu` с обратной связью в UI

5. **Телеметрия:**
   - [ ] Добавить в `TelemetryLogFrame`: `mag_x`, `mag_y`, `mag_z`, `heading_deg`
   - [ ] Отправка через WebSocket в реальном времени
   - [ ] Отображение heading в Web UI (компас/роза ветров)

6. **Тестирование:**
   - [ ] Unit-тесты: драйвер ICM‑20948, калибровка, AHRS с магнитометром
   - [ ] Интеграционный тест: стабильность yaw при длительном заезде
   - [ ] Полевой тест: сравнение heading с компасом/картой
   - [ ] Сравнение с LSM6DS3 + MMC5983MA (если оба варианта доступны)

**Зависимости:**
- Требуется поставка ICM‑20948
- Блокирует улучшение стабилизации yaw в длительных заездах
- Не блокирует базовую функциональность (можно работать с MPU‑6500 или LSM6DS3)

**Файлы для модификации:**
- `projects/rc_vehicle/firmware/common/icm20948_driver.hpp` (новый)
- `projects/rc_vehicle/firmware/common/icm20948_driver.cpp` (новый)
- `projects/rc_vehicle/firmware/common/madgwick_filter.hpp`
- `projects/rc_vehicle/firmware/common/vehicle_ekf.hpp`
- `projects/rc_vehicle/firmware/common/telemetry_log.hpp`
- `projects/rc_vehicle/firmware/esp32_common/imu_handler.cpp`

**Оценка сложности:** 4–7 дней (включая отладку на реальном устройстве)

**Примечание:** магнитометр в ICM‑20948 находится в том же корпусе что и IMU — это упрощает монтаж, но возможна интерференция от мотора/ESC. Требуется тщательное размещение и калибровка.

## TASK-RC-003 — Интеграция магнитометра MMC5983 (альтернатива/дополнение)
**Приоритет:** MEDIUM
**Статус:** [ ] В пути (ожидается 8 апреля)

**Цель:** добавить магнитометр MMC5983 как дополнение к LSM6DS3 (если ICM‑20948 не будет использоваться).

**Описание:**
Магнитометр MMC5983 обеспечивает измерение магнитного поля по 3 осям с высоким разрешением (до 0.1 мГн) и частотой обновления до 1000 Гц.

**Преимущества перед ICM‑20948:**
- Более высокая частота обновления магнитометра (1000 Гц vs 100 Гц)
- Можно разместить отдельно от IMU для минимизации помех
- Работает в паре с LSM6DS3

**Недостатки:**
- Два отдельных датчика вместо одного (сложнее монтаж)
- Требуется синхронизация данных между IMU и магнитометром

**Требуемые работы:** (аналогично TASK-RC-002, но для отдельного магнитометра)

1. **Аппаратная часть:**
   - [ ] Подключить MMC5983 к ESP32‑S3 по I2C или SPI
   - [ ] Разместить датчик как можно дальше от силовых проводов/мотора
   - [ ] Добавить в BOM

2. **Драйвер датчика:**
   - [ ] Создать класс `Mmc5983Driver` для ESP32‑S3
   - [ ] Реализовать инициализацию и чтение данных
   - [ ] Калибровка hard-iron / soft-iron
   - [ ] Проверка связи (регистр Product ID по адресу 0x2F, ожидаемое значение 0x30)

3. **Сенсорная фузия:**
   - [ ] Расширить `MadgwickFilter` для работы с магнитометром (9-DOF AHRS)
   - [ ] Добавить параметр `use_magnetometer`

4. **Калибровка:**
   - [ ] Вращение на 360° для калибровки магнитометра
   - [ ] Сохранение коэффициентов в NVS

5. **Телеметрия:**
   - [ ] Добавить `mag_x/y/z`, `heading_deg` в лог

6. **Тестирование:**
   - [ ] Unit-тесты и интеграционные тесты

**Зависимости:**
- Требуется закупка MMC5983MA
- Альтернатива или дополнение к TASK-RC-002 (ICM‑20948)

**Оценка сложности:** 3–5 дней

## Стратегия MVP

**Основной выбор для MVP:** ICM‑20948 (TASK-RC-002) — единственный 9-DOF в одном корпусе.
До его прихода (29 апр – 2 мая) использовать MPU‑6500.

**LSM6DS3** (TASK-RC-001) — если придёт раньше, запустить как промежуточный IMU и для сравнения с ICM‑20948.

**MMC5983MA** (TASK-RC-003) и **LSM6DS3+MMC5983MA** — альтернатива ICM‑20948 если тот окажется плох по магнитным помехам (можно разнести датчики физически). Решить после тестов.

**MPU‑9250** — только бенчмарк, не для MVP. ⚠️ Высокий риск подделки: снят с производства InvenSense, рынок наводнён клонами с несовместимыми регистрами. Перед использованием проверить WHO_AM_I (должен быть 0x71 для оригинала).
