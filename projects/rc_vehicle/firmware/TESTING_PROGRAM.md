# Программа испытаний и доведения RC-машинки

## Фаза 0 — Стендовые проверки (без движения)

- [ ] **IMU bias drift** — 5-10 мин неподвижности, записать телеметрию, оценить стабильность bias гироскопа
- [ ] **Failsafe** — отключить RC и Wi-Fi, убедиться что моторы встают в нейтраль за 250ms
- [ ] **Телеметрия** — подключиться по WebSocket, проверить 20 Hz стрим и 100 Hz лог в PSRAM, записать baseline шумов датчиков

## Фаза 1 — Базовое движение (DirectLaw)

- [ ] **Прямолинейный проезд** — режим DirectLaw, газ 30%, проехать 5м прямо. Оценить drift гироскопа и шум акселерометра в движении
- [ ] **Повороты на месте** — руль в крайнее положение, малый газ. Верифицировать yaw rate гироскопа
- [ ] **Торможение** — разгон до ~50% и сброс газа. Проверить ZUPT-детекцию EKF (переход в нулевую скорость)

## Фаза 2 — Калибровка положения IMU относительно центра масс

### Проблема

Акселерометр, смещённый от центра масс (CoM) на вектор **(rx, ry)**, измеряет дополнительные компоненты:

```
a_imu = a_com + α × r + ω × (ω × r)
```

- **α × r** — тангенциальное ускорение (при изменении угловой скорости)
- **ω × (ω × r)** — центростремительное ускорение (при вращении)

Без коррекции EKF получает искажённые данные, особенно в поворотах.

### Вариант B — Круговой тест (основной)

1. Машинка едет по кругу с фиксированным рулём и PID по скорости (~30% газ)
2. В установившемся режиме: α ≈ 0, ω = const
3. Центростремительная добавка на IMU: `a_centripetal_imu = ω² × r_imu_to_center`
4. На CoM: `a_centripetal_com = ω² × R` (R — радиус поворота CoM)
5. Разница даёт смещение: `a_imu - a_com = ω² × (r_offset)`
6. **Два круга** (по часовой и против часовой) — решаем систему для (rx, ry)

**Реализация:**
- Новая стадия в `CalibrationManager` — `CircularCalibration`
- Автоматический круговой проезд (фиксированный руль + PID по скорости, аналогично `auto_forward`)
- Сбор данных в установившемся режиме, фильтрация выбросов, расчёт (rx, ry)
- Сохранение в NVS, коррекция акселерометра в основном цикле

**Протокол:**
- [ ] Выбрать ровную площадку (~3×3м минимум)
- [ ] Круг по часовой: фиксированный руль, запись 10-15 сек установившегося режима
- [ ] Круг против часовой: аналогично
- [ ] Расчёт (rx, ry), сохранение в NVS
- [ ] Верификация: сравнение телеметрии до/после коррекции

### Вариант C — Онлайн-оценка через расширение EKF (альтернативный)

1. Расширить вектор состояния: `[vx, vy, r]` → `[vx, vy, r, rx, ry]`
2. При достаточном возбуждении (повороты разного радиуса) фильтр сам оценит смещение
3. **Преимущества:** не нужна отдельная процедура, адаптируется при изменении CoM (другая батарея/нагрузка)
4. **Риски:** offset наблюдаем только при ненулевых ω и α — может не сходиться при прямолинейном движении. Нужен анализ observability матрицы.

## Фаза 3 — Настройка фильтров

- [ ] **Madgwick β** — серия проездов с β = 0.05, 0.1, 0.2. Оценить latency ориентации vs уровень шума. Критерий: минимальный β при котором ориентация успевает за реальными манёврами
- [ ] **Butterworth LPF cutoff** — тест на грунте vs асфальте. Найти cutoff при котором фильтруется вибрация подвески, но сохраняется реальный yaw rate. Текущий default: 30 Hz
- [ ] **EKF верификация скорости** — проезд известного расстояния (мерная лента), замер времени, сравнение с интегралом EKF speed_ms. Оценить drift

## Фаза 4 — PID-настройка стабилизации

- [ ] **Yaw Rate PID (Normal mode)** — step response: резкий поворот руля на полном ходу. Записать overshoot и settling time yaw rate. Целевые значения: overshoot < 20%, settling < 200ms
- [ ] **Pitch compensation** — заезд на уклон (~10-15°). Убедиться что газ автоматически компенсируется, машина не скатывается
- [ ] **Slip Angle PID (Drift mode)** — на скользком покрытии (плитка, линолеум). Добиться стабильного контролируемого заноса с target_deg из конфига
- [ ] **Oversteer Guard** — в Normal mode спровоцировать начало заноса (резкий газ в повороте). Проверить что guard срабатывает и снижает газ

## Фаза 5 — Kids Mode и безопасность

- [ ] **Пресеты** — проверить каждый (Toddler/Child/Preteen): реальная max скорость, угол поворота, плавность разгона
- [ ] **Anti-spin** — скользкая поверхность, газ в пол. Убедиться что throttle снижается при обнаружении пробуксовки (slip > threshold)
- [ ] **Accel limiter** — резкий газ с нуля. Убедиться что разгон плавный (accel < threshold)
- [ ] **Комбинированный тест** — ребёнок (или имитация) управляет в каждом пресете. Субъективная оценка предсказуемости и безопасности

## Оборудование и площадки

- Ровная площадка ~5×5м (асфальт) — фазы 1-4
- Скользкое покрытие (плитка, линолеум) — Drift, Anti-spin
- Уклон ~10-15° — Pitch compensation
- Мерная лента 5-10м — верификация EKF
- Ноутбук с браузером — Web UI телеметрия в реальном времени
- Полностью заряженная батарея + батарея на ~50% — проверка поведения при разном заряде
