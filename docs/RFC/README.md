# RFC — Request for Comments

Документирование RFC для предложений новых сервисов, архитектурных решений, и крупных изменений.

---

## Структура

```
docs/RFC/
├── README.md                    # Этот файл
├── rfc-template.md             # Шаблон для нового RFC
├── rfc-0001-service-name.md    # Примеры реализованных RFC
├── rfc-0002-another.md
└── ...
```

---

## Процесс

### 1. Создание

1. Скопируй `rfc-template.md` в `rfc-XXXX-description.md` (где XXXX — следующий номер)
2. Заполни обязательные секции:
   - **Резюме** — что и зачем?
   - **Мотивация** — какие проблемы решает?
   - **Дизайн** — как реализовано?
   - **План** — stages и timeline
3. Статус: **Draft**

### 2. Review

1. Создай PR с RFC документом
2. Обсуди с командой (Slack / код-ревью)
3. Обновляй RFC на основе feedback
4. Статус: **Review**

### 3. Approval

1. Получи sign-off от техлида
2. Merge PR в `develop`
3. Статус: **Approved**

### 4. Implementation

1. Создавай задачи/issues для реализации (ссылка на RFC)
2. Обновляй RFC при изменениях во время разработки
3. Статус: **Implemented** (когда merged в main)

### 5. Closure

- **Implemented:** RFC успешно реализован, документация актуальна
- **Rejected:** Решили не реализовывать (с объяснением в комментариях)

## Названия

Нумерация последовательная: `rfc-0001.md`, `rfc-0002.md`, ...

Формат имена файла: `rfc-XXXX-short-description.md`

Пример:
- `rfc-0001-telemetry-aggregation-service.md`
- `rfc-0002-audit-logging-system.md`
- `rfc-0003-websocket-optimization.md`

## Секции шаблона

| Секция | Обязательна? | Описание |
|--------|------------|---------|
| Резюме | ✓ | 2-3 предложения |
| Мотивация | ✓ | Почему это нужно |
| Дизайн сервиса | ✓ | API, схема, интеграция |
| Реализация | ✓ | Структура кода, миграции |
| Тестирование | ✓ | Unit/integration/load tests |
| Развёртывание | ✓ | Docker, миграции, мониторинг |
| Альтернативы | ✓ | Почему этот подход? |
| Ограничения | ✗ | Если есть tech debt |
| План реализации | ✓ | Phases, timeline |
| Ссылки | ✗ | Related ADR, issues |

## Советы

- **Будь конкретен:** вместо "оптимизируем" — "уменьшим latency с 500 мс до 100 мс"
- **API-first:** сначала дизайни API, потом реализацию
- **Рассмотри нагрузку:** сколько RPS? Какая data retention?
- **Think operations:** как мониторить, алертить, дебажить?
- **Думай о миграции:** как обновить систему без downtime?
