## Продумать создание датчиков (возможно тут лучше подойдет вычисляемых/физических величин). Создать RFC

**Приоритет:** MEDIUM
**Статус:** [ ] Подготовка

Некоторые значения нужно вычислять и сохранять, например при проведении эксперимента в трубе считать число M. Или например расход воздуха. В первом случае величина вычисляется с помощью формулы из начений датчиков (или других величин), во втором слуае будет сложнее, нужно считать интеграл по значениям и периодически сбрасывать значения, например между пусками.
По реализации:

Подумать как задавать алгоритмы вычесления этих данных, варианты:
1. простые формулы (многочлен, арифметические формулы и т.д.). усложнение расчета интегральных величин
2. добавить интрпритатор языка, в котором уже можно получать доступ не только к последним значениям, но и рядам значений. плюсы: максимальная гибкость. минусы: усложнение реализации, оператору необходимо знать этот язык

Делать внутри определенного сервиса или создавать новый, который будет получать данные и расчитывать значения. нужно ли добавлять стек например с c++ и uservices для скорости расчета, возможно тогда переписать telemetry-ingest-service на этот стек, так как кажется это самый нагруженный сервис.

**Польза:** Можно выводить и сохранять полезные данные

Пока не продумана

## Сделать RFC для алертов по датчикам

**Приоритет:** LOW
**Статус:** [ ] Подготовка

Есть идея реализовать алерты по значениям датчиков. При проведении экспериментов нужно наблюдать за значениями датчиков, бывают критичные значения при которых требуется вмешательство в эксперимент операторов. Но на графиках это можно пропустить, нужно иметь возможность добавлять в телеметрию панели с лампочками (зеленый, желтый, красный или настраиваемый цвет в зависимости от значения), цыфровыми значениями и текстом (возможно так же добавление звуковых сигналов). Так же это может потребовать переработать страницу телеметрии, добавить возможность сохранять состояние панелей в базе данных, и так же добавлять несколько экранов телеметрии (открывать в разных окнах браузера разные экраны). Так же можно при изменении страницы телеметрии можно добавить возможность добавлять стрелоные приборы по датчикам. Подумать про какие приборы можно еще добавить для отображения.

**Польза:** помогает администратору видеть состояние экспериментальной системы и вовремя реагировать на изменение ситуации.

**Возможная реализация:**
Пока не продумана

## Сделать RFC для систем управления экспериментом

**Приоритет:** LOW
**Статус:** [ ] Подготовка

Нужно продумать систему управлением экспериментом, запрос на такую систему довольно большой, но реализацию нужно будет делать уже после алертов по датчикам. Потонцеальные проблемы: отставание данных не тоько из-за датчиков и обработки, но и из-за присутствия системы (запросы в базу данных межсервисное взаимодействие), нужно будет оценить как этом может сказаться на управляющих сигналах, и попроги применимости системы (для систем с медленными изменениями фазавая задержка может быть не критичной).

**Польза:** помогает администратору видеть состояние экспериментальной системы и вовремя реагировать на изменение ситуации.

**Возможная реализация:**
